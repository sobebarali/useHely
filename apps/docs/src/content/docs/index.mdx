## Features

<CardGrid stagger>
	<Card title="Multi-Tenant Architecture" icon="setting">
		Isolated data per hospital with schema-per-tenant design. Each hospital operates independently with secure data isolation.
	</Card>
	<Card title="Role-Based Access Control" icon="approve-check">
		Pre-defined roles (Admin, Doctor, Nurse, Pharmacist, Receptionist) with customizable permissions and fine-grained ABAC policies.
	</Card>
	<Card title="Patient Management" icon="document">
		Complete patient lifecycle from registration to discharge. Support for OPD and IPD workflows with comprehensive records.
	</Card>
	<Card title="Appointment Scheduling" icon="list-format">
		Doctor availability, appointment booking, queue management, and real-time status tracking.
	</Card>
	<Card title="Prescription & Pharmacy" icon="pencil">
		Digital prescriptions, pharmacy dispensing workflows, inventory management, and stock tracking.
	</Card>
	<Card title="Reports & Analytics" icon="open-book">
		Comprehensive reporting for patients, appointments, prescriptions, and operational metrics.
	</Card>
</CardGrid>

## User Roles

| Role | Primary Functions |
|------|-------------------|
| **Hospital Admin** | Hospital configuration, user management, departments, reports |
| **Doctor** | Patient consultations, prescriptions, appointments |
| **Nurse** | Patient care, vitals recording, medication administration |
| **Pharmacist** | Prescription dispensing, inventory management |
| **Receptionist** | Patient registration, appointment scheduling, check-in |

## Tech Stack

| Layer | Technologies |
|-------|--------------|
| **Frontend** | React, TanStack Router, shadcn/ui, TailwindCSS |
| **Backend** | Node.js, Express.js, Better-Auth |
| **Database** | MongoDB Atlas, Redis |
| **Infrastructure** | Docker, Nginx, GitHub Actions |
